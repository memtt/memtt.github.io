NUMAPROF

    I currently provide a first version of NUMAPROF, and, of course, I have some ideas on what to improve but don't have the time to do it myself now. If you are interested to contribute you can find here some starting points which might motivate you. I will be pleased to provide help, answer questions, detailed ideas, code reviews and to merge your great stuff. Be also welcome to provide your own ideas.

    You can join by looking on the Github repositoy.

    Core features

    Call stacks

    Add support of call stacks in instrumentation instead of the uniq call site. This require a lot of work but is an important feature.

    Translation

    Could be really cool to get translations for the GUI and command-line tool.

    Time charts

    Add some metric over time (NUMA node accesses).

    NUMA distance

    Report the NUMA distance instead of just local/remote. First globally then for each callsite.

    Improve help

    Ideally, I wanted to get documentation of all the features provided by the interface linked to some simple example of bad pattern it helps to detect.

    Integration with other tools

    Valgrind

    Pintool looks more performant than valgrind but for portability it might be interesting to also have integration in with this tool.

    DynamoRIO

    Similar to pintool but with support of ARM. It might be interesting to compare the final instrumentation performance.

    Maqao

    This time with static instrumentation, but still interesting. It might require some work to instrumentation the libraries.

    Curses GUI

    Could be interesting to also get a curses GUI python.

    Electron

    Provide a standard graphical interface by using electron.
