MALT

    A memory profiling tool to track malloc
    Find where you allocate your memory and how, reduce the memory footprint of your applications and limit overhead of memory management.
    Source - 1.0.0
    Others

    MALT is a memory profiling tool used to find where you allocate your memory and how. It helps in using the same approach as valgrind / kcachgrind for performance, but, to reduce your memory footprint and performance impact of memory management.

    It is the first published version. Of course there are plenty of things to fix. Thank you for your understanding and to report bugs. Feel free to contribute to improve it.

    Features

    Code annotation

    MALT provides code annotations to map allocation metrics directly onto your source code. It also uses human readable units (KB, GB...).

    Thread statistics

    You get some statistics about usage of memory functions over threads.

    Time profiles

    It provides some metrics over time, like the number of allocation, allocation rates, usage of virtual, physical and requested memory.

    Track global variables

    In some applications (e.g. in FORTRAN), global variables or TLS can use a large amount of memory. MALT also tracks them for all linked libraries

    Allocation properties

    In addition to memory size MALT also give you some allocation properties, like : minimal, mean and maximal lifetime of chunks linked to call sites.

    Call stacks

    Memory allocations are mapped onto call stacks and call sites.

    You can get a more complete list of features on this page
    MALT instrument you program by wrapping the malloc (and free, calloc...) thanks to Unix symbol overriding through usage of LD_PRELOAD. Additionally, it uses backtrace() to reconstruct the call stack on each call to memory functions.
    MALT provides a web GUI based on new technologies like NodeJS, AngularJS, D3JS. Thanks to this it might be easier for you to run MALT on a remote node by forwarding the connection via SSH. 


    malt ./my_program

malt-webview \
	-i ./malt-my_program-1588.json

browser http://localhost:8080
    MALT is used by running the profiling tool on your command just as you do with valgrind. Then, open the profile file with the MALT webserver and connect to it with your browser by using http://localhost:8080.
